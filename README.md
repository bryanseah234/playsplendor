# Splendor (Java Implementation)

A modular, strictly MVC-based implementation of the board game Splendor in Java.

## Features
- **MVC Architecture:** Separation of concerns between Model, View, and Controller.
- **Custom Exception Handling:** Robust error management using `SplendorException`.
- **Configurable:** Game rules and setup parameters loaded from `src/resources/config.properties`.
- **Console Interface:** 
  - **ASCII Dashboard:** Cards and game state are rendered in a clean, frame-based dashboard.
  - **Color Support:** Gems and player info are color-coded (Red, Green, Blue, White, Black, Gold).
  - **Smart Menus:** Options are dynamically enabled/disabled based on game state (e.g., you can't buy if you can't afford).
  - **Interactive Prompts:** Intuitive sub-menus for selecting gems and cards.
  - **Undo Feature:** Allows players to undo their last turn by typing `Z` or `UNDO`, implemented with the Memento design pattern to perform a full rollback of the game state (including cards, gems, reserves, and board decks).
- **Network Support (Bonus):** Multiplayer capabilities via TCP sockets.

## Prerequisites
- Java JDK 17 or higher.
- A terminal that supports ANSI color codes (most modern terminals do).

## Building the Project

### Windows
Run the provided batch script:
```batch
compile.bat
```

### Linux / macOS
Run the shell script:
```bash
chmod +x compile.sh
./compile.sh
```

## Running the Game

### Console Mode (Default)
To play the game in the console:

**Windows:**
```batch
run.bat
```

**Linux / macOS:**
```bash
chmod +x run.sh
./run.sh
```

### Server Mode (Network Play)
To start the game server for multiplayer:

**Windows:**
```batch
java -cp classes com.splendor.Main --server
```

## Project Structure
```
src/com/splendor/
├── config/       # Configuration management (IConfigProvider)
├── controller/   # Game logic orchestration (GameController)
├── exception/    # Custom exceptions (SplendorException)
├── model/        # Core game entities (Game, Player, Board)
├── network/      # Network handling (ServerSocketHandler)
├── util/         # Utility classes (InputResolver, GameLogger)
└── view/         # User interface (ConsoleView, IGameView)
```

## Configuration
Game settings can be modified in `src/resources/config.properties`:
- **Winning Points:** `game.points.win=15`
- **Max Tokens:** `game.max_tokens=10`
- **Player Scaling:** Token counts per player count.
